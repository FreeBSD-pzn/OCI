Краткие инструкции.

В данном случае используется операционная система FreeBSD:

$ uname -a <Enter>
FreeBSD sony 11.3-RELEASE FreeBSD 11.3-RELEASE #0 r349754: Fri Jul  5 04:31:33 UTC 2019
root@releng2.nyi.freebsd.org:/usr/obj/usr/src/sys/GENERIC  i386

Terraform используется из ports [/usr/ports/sysutils/terraform]
$ terraform -v
Terraform v0.12.3

Для использования данной версии terraform введена проверка версии API OCI
в файле provider.tf и продублирована переменная в файле variable.tf

* для дополнительной информации:
* https://docs.oracle.com/en-us/iaas/Content/API/SDKDocs/terraformconfigurationupgrades.htm
*

Перед использованием Вам необходимо зарегистрироваться в OCI и создать
каталог ~/.oci в своем домашнем каталоге. В этот каталог необходимо
поместить файл, содержащий PRIVATE KEY, который необходимо сохранить
из данных аккаунта при регистрации в OCI.

Желательно начать всего с двух файлов: provider.tf и variable.tf
Файл variable.tf создается из variable.tf.template заполняя необходимые
данные:
tenancy_ocid - необходимо взять из вашего аккаунта OCI
user_ocid    - необходимо взять из вашего аккаунта OCI
fingerprint  - необходимо взять из вашего аккаунта OCI
compartment_ocid - необходимо взять из вашего аккаунта OCI
region - необходимо взять из вашего аккаунта OCI

private_key_path - указать путь и имя файла, который был размещен в ~./oci

После этого просто запустите: terraform init

Если все прошло без ошибок, то можно в используемый каталог скопировать
остальные файлы, создать пары security key для удаленного входа в
виртуальные машины ( ~/.ssh/vm-german-01.pub и ~/.ssh/vm-german-02.pub )
и запустить:

$ terraform plan

и затем

$ terraform apply

После выполнения должны быть созданы 2-е виртуальные машины и выведены IP
адреса этих виртуальных машин, вход в которые можно проверить с помощью ssh
ssh -i ~/.ssh/vm-german-01 username@IP-vm-german-01
ssh -i ~/.ssh/vm-german-02 username@IP-vm-german-02

имена пользователей:
для операционной системы Ubuntu - ubuntu
для операционной системы CenOS  - opc

Пояснения:

Образы операционных систем отобраны для региона -> eu-frankfurt-1
если вами был выбран другой регион, то необходимо изменить идентификаторы
образов виртуальных машин в соответствии с выбранным регионом.

Адрес списка идентификаторов образов виртуальных машин указан в начале
файла images.tf

08.01.2022
Когда создается VCN в OCI в root или в compartment, то OCI всегда по умолчанию
создает таблицу маршрутизации (oci_core_default_route_table) и списки безопасности
(oci_core_default_security_list), поэтому если вами будут созданы свои ресурсы, то
в панели управления будут присутствовать по две версии указанных ресурсов.
Чтобы избежать дублирования ресурсов можно использовать созданные OCI по умолчанию
указанные ресурсы, это и было выполнено в новой версии файла network.tf.
Предыдущая версия network.tf была переименована в network.tf.work.

Подробно ознакомиться с создаваемыми OCI ресурсами по умолчанию можно здесь:
https://docs.oracle.com/en-us/iaas/Content/API/SDKDocs/terraformbestpractices_topic-vcndefaults.htm
